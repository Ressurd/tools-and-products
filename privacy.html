Privacy Policy for Tab Session Manager, Quick Tab Cleaner, and Link Inbox.

Last updated

May 10, 2026

What data is stored

Tab Session Manager stores data that is needed to save and restore tab sessions created by the user. Depending on how the user uses the extension, this may include:

  • Session titles entered or edited by the user
  • Tags and notes added by the user
  • Tab titles, tab URLs, and favicon URLs for saved sessions
  • Timestamps related to saved or updated sessions
  • Window counts and tab counts for saved sessions

How data is used

Stored data is used only to provide the extension's session management features. These features include saving the current window or all open windows, showing saved sessions, searching saved sessions, editing or deleting saved sessions, restoring sessions in a new window, and restoring sessions in the current window.

The extension also supports user-controlled JSON export and import so users can create or restore their own local backups.

Where data is stored

Tab Session Manager stores saved session data in the user's local browser storage using chrome.storage.local.

The extension does not use cloud sync, does not send saved session data to a remote backend, and does not provide a server-side account system. If a user exports a JSON backup, that exported file is created because the user chose to export it and is handled by the user outside the extension's local storage.

Permissions used

  • tabs: Used to read the currently open tabs and windows when the user saves a session, and to restore saved tabs when the user chooses a restore action.
  • storage: Used to save, load, edit, delete, export, and import saved session data in local browser storage.

What is not collected / not used

  • No cloud sync is currently provided.
  • No analytics are currently collected by the extension.
  • No advertising identifiers are collected or used.
  • No remote backend is currently used for saved session data.
  • No saved session data is sold.
  • No user account or login system is currently provided.

Quick Tab Cleaner

Quick Tab Cleaner is a Chrome extension that helps users review currently open tabs, find exact duplicate URLs, preview duplicate groups, and safely close duplicate tabs after a user action.

Quick Tab Cleaner keeps pinned tabs open and keeps active tabs first when possible. It does not close duplicate tabs automatically in the background.

Quick Tab Cleaner data read

Quick Tab Cleaner reads information about currently open Chrome tabs only as needed to calculate and display duplicate tab groups in the popup. This may include:

  • Tab title
  • Tab URL
  • Domain
  • Window id
  • Tab index
  • Active tab state
  • Pinned tab state

Quick Tab Cleaner data use

Tab data is used locally in the popup to find duplicate tabs by exact URL, show duplicate group previews, show domain summaries, mark which tabs will stay open, and mark which duplicate tabs will close.

Duplicate tabs are closed only after the user chooses a cleanup action, such as cleaning one duplicate group or closing duplicates across all groups.

Quick Tab Cleaner storage

Quick Tab Cleaner does not store tab data in extension storage.

The current tab snapshot is processed locally while the popup is open. The extension does not send tab titles, URLs, domains, or tab state to a remote backend.

Quick Tab Cleaner permissions used

  • tabs: Used to read currently open tab metadata, find exact URL duplicates, show duplicate previews, identify active and pinned tabs, and close duplicate tabs after the user chooses a cleanup action.

Quick Tab Cleaner what is not collected / not used

  • No cloud sync is currently provided.
  • No analytics are currently collected by the extension.
  • No advertising identifiers are collected or used.
  • No remote backend is currently used.
  • No tab data is stored in extension storage.
  • No tab data or personal data is sold.
  • No automatic background cleanup is performed.

Link Inbox

Link Inbox is a Chrome extension that lets users save the current tab to a local link inbox, manage links as an Inbox / Done queue, search saved links by title or URL, and create or restore JSON backups.

Link Inbox stores the current tab only when the user clicks Save current tab. It does not automatically collect links in the background.

Link Inbox data read and stored

Link Inbox reads the current tab title, URL, and favicon URL for the popup preview and for the user-triggered save action.

Saved Link Inbox records may include:

  • id
  • title
  • url
  • faviconUrl
  • savedAt
  • status

Link Inbox data use

Link Inbox uses saved data only to provide the local link queue features selected by the user. These features include showing saved links, marking links as done, moving done links back to the inbox, opening saved links, deleting saved links, searching by title or URL, exporting a JSON backup, and importing a JSON backup.

During import, Link Inbox uses exact URL matching to skip duplicate saved links instead of adding the same URL twice.

Link Inbox storage

Link Inbox stores saved link data only in the user's local browser storage using chrome.storage.local.

JSON export and import are controlled by the user. The extension does not send saved links, tab titles, URLs, or favicon URLs to a remote backend.

Link Inbox permissions used

  • tabs: Used to read the current active tab title, URL, and favicon URL for the popup preview and user-triggered save action, and to open saved links when the user chooses the Open action.
  • storage: Used to save, load, update, delete, export, and import Link Inbox data in local browser storage.

Link Inbox what is not collected / not used

  • No automatic link collection is performed.
  • No full browsing history scan is performed.
  • No page content is collected.
  • No cloud sync is currently provided.
  • No analytics are currently collected by the extension.
  • No advertising identifiers are collected or used.
  • No remote backend is currently used.
  • No saved link data or personal data is sold.
